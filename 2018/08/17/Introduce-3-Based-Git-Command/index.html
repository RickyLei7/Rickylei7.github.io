Introduce 3 Based Git Command

Posted on 2018-08-17

1. git init


The git init command creates a new Git repository. it can be used to convert an existing, unversioned project to a Git repository or initialize a new, empty repository.

2. git add


The git add command adds a change in the working directory to the staging aera. It will tells Git that you want to updates a file in the next commnit.
The git add . means stages new and modifiled, without deleted.

3. git commit -v


The git commit command captures a snapshot of the project’s currently staged changes. Committed snapshots can be thought of as “safe” versions of a project—Git will never change them unless you explicitly ask it to. Prior to the execution of git commit, The git add command is used to promote or ‘stage’ changes to the project that will be stored in a commit. These two commands git commit and git add are two of the most frequently used.

-v

–verbose

Show unified diff between the HEAD commit and what would be committed at the bottom of the commit message template to help the user describe the commit by reminding what changes the commit has.
