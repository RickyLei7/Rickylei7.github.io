Difference of primitives type and complex type

In JavaScript, ECMA stander defin 6 primitives type: number, string, boolean, null, undefined, symbol, and 1 complex type: object. In the computer, the primitives store in the Stack, the complex store in the Heap. but address of heap store in the Stack.

The data in the Stack is first in first out, and the Heap is unordered.

Primitives type assignment is only deep copy, it will create a new stack. Shallow copy is copy the address of the heap. And complex type are deep copy and shallow copy.

Garbage collection：If an object no reference. It is a garbage. Will be collection by browser. when the garbage collection depends browser.
