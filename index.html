Prototype and Prototype Chain

Posted on 2018-10-29

Prototype and Prototype Chain

JavaScript is difference with Java or C++ object-oriented language. JavaScript doesn’t have class or inheritance. Forget class and inheritance, they are not in JavaScript.

Prototype and Prototype Chain

When it comes to inheritance, JavaScript only has one construct: object. Each object has a private property call prototype, which holds a link to another object.

When we try to fine a property in the object, if current object has not have the property. It will try to find in the prototype of object. If it still can’t find it, will try next prototype of prototype object. This process of lookup as a chain. We call it Prototype Chain.

Until lookup to the top prototype object. This is Object.prototype, which includes the most common method as toString, valueOf, hasOenProperty. This is why we can using these method in the any object.

Difference of primitives type and complex type

Posted on 2018-10-28

Difference of primitives type and complex type

In JavaScript, ECMA stander defin 6 primitives type: number, string, boolean, null, undefined, symbol, and 1 complex type: object. In the computer, the primitives store in the Stack, the complex store in the Heap. but address of heap store in the Stack.

The data in the Stack is first in first out, and the Heap is unordered.

Primitives type assignment is only deep copy, it will create a new stack. Shallow copy is copy the address of the heap. And complex type are deep copy and shallow copy.

Garbage collection：If an object no reference. It is a garbage. Will be collection by browser. when the garbage collection depends browser.

HTTP Introduction and Curl Command

Posted on 2018-08-18 | Edited on 2018-08-20

The Hypertext Transfer Protocol (HTTP) is an application-level protocol for distributed, collaborative, hypermedia information system. HTTP is all Web Developer must have a strong understanding of this protocol.

HTTP Requests

An HTTP clients sends an HTTP request to a server. Including 4 part.

  1. verb path protocl and version
    Ex:GET /HTTP 1.1
    Verb：GET POST HEAD PUT DELETE TRACE OPTIONS.

  2. Request Header
    stander format
    Key1: value1
    Key2: value2
    Key3: value3
    The Request Header including the client infomation and Request content, the Content-Type define the format of fouth part.

  3. Enter mark.
    a blank line between the Request Header and Request Content. shows that the Request Header was finish, the following is Request Content.

  4. Request Content
    Request Content include the infomation which by client commited.

Cheaking HTTP Request by Chrome devoloper tool.
Open the devoloper tool on the Chrome browser, selet Network, next tyep the address in the address box. Click the view source again! Click the view source.

HTTP Response

After receiving and interpreting a request message, a server responds with an HTTP respons. Including 4 part.

  1. protocol/version status-code status
    ex: HTTP/1.1 200 OK

####status Code####
The Status-Code element is a 3-digit integer where first digit of the Status-Code defines the class of response and the last two digits do not have any categorization role. There are 5 values for the first digit:
1xx: Informational
It means the request was received and the process is continuing.
2xx: Success
It means the action was successfully received, understood, and accepted.
3xx: Redirection
It means further action must be taken in order to complete the request.
4xx: Client Error
It means the request contains incorrect syntax or cannot be fulfilled.
5xx: Server Error
It means the server failed to fulfill an apparently valid request.

  1. Response Header
    Format:
    Key1: value1
    Key2: value2
    Content-Type: text/html; charset=utf-8
    text/html means the fourth part is HTML

  2. Enter mark.
    Similay with request. a blank line between the Response Header and Response Content.

  3. Response Content
    Shows that what content the browser needs to download.

1. git init


The git init command creates a new Git repository. it can be used to convert an existing, unversioned project to a Git repository or initialize a new, empty repository.

2. git add


The git add command adds a change in the working directory to the staging aera. It will tells Git that you want to updates a file in the next commnit.
The git add . means stages new and modifiled, without deleted.

3. git commit -v


The git commit command captures a snapshot of the project’s currently staged changes. Committed snapshots can be thought of as “safe” versions of a project—Git will never change them unless you explicitly ask it to. Prior to the execution of git commit, The git add command is used to promote or ‘stage’ changes to the project that will be stored in a commit. These two commands git commit and git add are two of the most frequently used.

-v

–verbose

Show unified diff between the HEAD commit and what would be committed at the bottom of the commit message template to help the user describe the commit by reminding what changes the commit has.
